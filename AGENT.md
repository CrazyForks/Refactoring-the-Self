# AGENT.md — 连载项目规范

## 项目文件

- `README.md` — 项目首页 + 连载目录
- `ABOUT.md` — 作者介绍
- `AGENT.md` — 连载项目规范
- `chapters/` — 章节文件目录

## 文件命名

- 格式：`{序号}-{英文-slug}.md`
- 序号两位，从 `01` 开始递增
- slug 用英文短横线连接，概括章节核心意象
- 示例：`03-four-hundred-ducks.md`

## 标题结构

```markdown
# 主标题（核心事件/意象）

## —— 副标题（情感落脚点）
```

- 副标题以 `## ——` 开头，破折号为全角
- 第一章可以没有副标题

## 章节状态

两种状态：

- **已发布**：完整正文 + 底部导航
- **预告**：标题结构 + 预告块引用 + 底部导航，正文待撰写

预告格式：
```markdown
> 预告：这一章仍在路上。
>
> （2-3 句悬念式摘要）
```

## 底部导航

每章末尾统一格式：

```markdown
---

[← 上一章：{副标题}](./XX-slug.md) | [返回目录](../README.md) | [下一章：{副标题} →](./XX-slug.md)
```

- 导航中用章节的**副标题**（`## ——` 后的内容），第一章无副标题则用主标题摘要
- 第一章不写"上一章"，用 `[← 返回目录](../README.md)` 代替
- 最新一章不写"下一章"的链接
- 预告页同样需要导航
- 链接使用相对路径

## README 目录

```markdown
👉 [阅读 第{XX}章：《标题》](./chapters/XX-slug.md)

👉 [阅读 第{XX}章：《标题》（预告）](./chapters/XX-slug.md)
```

- 每行一条，章节间空行隔开
- 预告章节标注 `（预告）`
- 按章节号升序排列
- 对章节标点符号错误进行修正

## 操作流程

### 处理章节（核心流程）

处理一个章节时，按以下顺序执行：

**① 读取旧文件（如有）**
1. 先读取旧中文名文件的全部内容
2. 确认其中包含的是**完整正文**还是**简短摘要**
3. ⚠️ 关键：不要跳过这一步就直接删旧文件！必须先读内容

**② 确认用户意图**

**在执行任何操作之前，先问用户**：这章要"发布"还是"预告"？

如果用户已明确说"发布"或"预告"，按用户说的做。如果用户没有明确说（比如只说"处理"、"执行"、"按照提示词改"），**必须问清楚**，不得自行推测。

> ⚠️ 规则：默认先问，绝不猜测。宁可多问一句，不要做错再改。

**③ 处理简体版**
1. 创建或更新 `chapters/{序号}-{slug}.md`
2. 写入规范的标题结构（主标题 + 副标题）
3. 根据判断结果写入：
   - **已发布**：写入完整正文 + 底部导航
   - **预告**：只预告块引用 + 底部导航，**不得写入完整正文**
4. **最后**删除旧中文名文件（确认新文件已正确创建后再删）

预告摘要提取规则：
> - 从原文中提取**核心悬念、冲突或转折点**
> - 不要直接复制开头或结尾，要留白
> - 每句单独一行，用 `>` 包裹
> - 预告块引用中不放入完整正文
> - 示例：
>   ```markdown
>   > 预告：这一章仍在路上。
>   >
>   > 他们说，拆开不难，装回去才难。
>   >
>   > 那个下午，我第一次摸到了一台发动机的心脏。
>   ```
> 
> ⚠️ 关键：预告页一定不要写完整正文！只写悬念摘要。

**④ 更新导航**
1. 上一章的 `下一章 →` 链接指向当前章
2. 当前章导航含 `← 上一章` 和 `返回目录`
3. 当前章是最新的 → 不写 `下一章 →`；后续有已发布章节 → 补上

**⑤ 更新 README 目录**
- 已发布：`👉 [阅读 第{XX}章：《标题》](./chapters/XX-slug.md)`
- 预告：`👉 [阅读 第{XX}章：《标题》（预告）](./chapters/XX-slug.md)`
- 按章节号升序排列

**⑥ 繁体同步**
- 简体版处理完成后，执行繁体转换（方法见下文）

### 批量处理（用户同时发两个章节时）

用户发两个旧中文名文件时，按以下固定规则处理：

- **第一个文件** — 预告更新：将其完整正文填充到当前 `chapters/` 中已有的预告页中，变更为"已发布"
- **第二个文件** — 新章节：创建为"预告"（预览格式），不写入完整正文

按顺序执行：先处理第一个（填充正文 → 已发布），再处理第二个（新文件 → 预告）。

## 繁体中文同步

### 基本原则

- **简体版本完成后，再做繁体同步**，顺序不可颠倒
- 使用 OpenCC 的 `s2hk.json`（简→繁香港）配置
- 繁体版放在独立目录 `chapters-hant/` 下
- 首页 README 和作者页也同步繁体版本

### 操作流程

1. 确认简体版所有修改已完成并保存
2. 运行转换命令：
   ```bash
   mkdir -p chapters-hant
   for f in chapters/*.md; do
     opencc -i "$f" -c s2hk.json -o "chapters-hant/$(basename "$f")"
   done
   ```
3. 分别转换首页和作者页：
   ```bash
   opencc -i README.md -c s2hk.json -o README-zh-Hant.md
   opencc -i ABOUT.md -c s2hk.json -o ABOUT-zh-Hant.md
   ```
4. 检查繁体版中的链接路径是否指向 `chapters-hant/`（无需手动改，OpenCC 只转换字符）
5. 在 README-zh-Hant.md 顶部添加转换说明：
   ```markdown
   > 本文由 OpenCC 自动从简体中文转换为繁体中文（香港）。
   > 如有表达不自然之处，欢迎反馈。
   ```

### 注意事项

- OpenCC 仅做字符转换，不做词汇本地化（如"软件"不会转为"軟件"）
- 导航链接中的路径无需转换，保持相对路径不变
- 繁体版不单独维护正文内容，每次更新简体版后重新转换
