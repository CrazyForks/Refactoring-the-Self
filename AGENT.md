# AGENT.md — 连载项目规范

## 项目文件

- `README.md` — 项目首页 + 连载目录
- `ABOUT.md` — 作者介绍
- `AGENT.md` — 连载项目规范
- `chapters/` — 章节文件目录

## 文件命名

- 格式：`{序号}-{英文-slug}.md`
- 序号两位，从 `01` 开始递增
- slug 用英文短横线连接，概括章节核心意象
- 示例：`03-four-hundred-ducks.md`

## 标题结构

```markdown
# 主标题（核心事件/意象）

## —— 副标题（情感落脚点）
```

- 副标题以 `## ——` 开头，破折号为全角
- 第一章可以没有副标题

## 章节状态

两种状态：

- **已发布**：完整正文 + 底部导航
- **预告**：标题结构 + 预告块引用 + 底部导航，正文待撰写

预告格式：
```markdown
> 预告：这一章仍在路上。
>
> （2-4 句悬念式摘要）
```

## 底部导航

每章末尾统一格式：

```markdown
---

[← 上一章](./XX-slug.md) | [返回目录](../README.md) | [下一章 →](./XX-slug.md)
```

- 第一章不写"上一章"，最新一章不写"下一章"的链接
- 预告页同样需要导航
- 链接使用相对路径

## README 目录

```markdown
👉 [阅读 第{XX}章：《标题》](./chapters/XX-slug.md)

👉 [阅读 第{XX}章：《标题》（预告）](./chapters/XX-slug.md)
```

- 每行一条，章节间空行隔开
- 预告章节标注 `（预告）`
- 按章节号升序排列
- 对章节标点符号错误进行修正

## 操作流程

### 新建预告页

1. 删除旧中文名文件（如有）
2. 创建 `chapters/{序号}-{slug}.md`
3. 写入标题结构 + 预告块引用 + 底部导航
4. 更新上一章的 `下一章 →` 链接
5. README 末尾追加目录条目（标注预告）

### 预告转发布

1. 用完整正文替换预告块引用
2. README 中去掉该章节的 `（预告）` 标记
3. 正文末尾可酌情加 1-2 句收尾段

### 新增章节时需同步的文件

- 章节文件本体
- 上一章的导航链接
- README 目录
- 下一章的导航（如有）
